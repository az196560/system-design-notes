# Embedding 与向量检索基础

---

## 什么是 Embedding

Embedding 是将离散对象（文字、图片、用户等）映射到连续的高维向量空间中的技术。语义相似的对象在向量空间中距离更近。

```
"猫" → [0.12, -0.45, 0.78, ..., 0.33]  (1536 维)
"狗" → [0.15, -0.42, 0.81, ..., 0.31]  (与"猫"距离近)
"汽车" → [-0.67, 0.23, -0.11, ..., 0.89] (与"猫"距离远)
```

---

## 文本 Embedding 模型

### Word2Vec（2013）

**CBOW：** 用上下文词预测中心词。
**Skip-gram：** 用中心词预测上下文词。

局限：一个词只有一个固定向量，无法处理一词多义（如"苹果"可以是水果或公司）。

### 句子/文档 Embedding

**Sentence-BERT：** 用 BERT 编码句子，取 [CLS] Token 或平均池化作为句子向量。Siamese 网络结构，对比学习训练。

**现代 Embedding 模型：** text-embedding-3（OpenAI）、E5、BGE 等。
- 对比学习（Contrastive Learning）：正样本对（相似文本）距离近，负样本对距离远
- 指令感知（Instruction-aware）：可以通过指令指定 Embedding 用途（如"为检索任务编码"vs"为分类任务编码"）

### Matryoshka Embedding

训练时同时优化多个维度前缀（如前 256 维、512 维、1536 维都有效），推理时可按需截断。低维做粗筛，高维做精排。

## 距离度量

### 余弦相似度（Cosine Similarity）

```
cos(A, B) = A·B / (‖A‖ × ‖B‖)
```

- 范围：[-1, 1]，1 表示方向相同，0 表示正交，-1 表示方向相反
- 只关注方向，不关注大小（长度归一化）
- **文本 Embedding 的默认选择**

### 欧氏距离（L2 Distance）

```
L2(A, B) = √(Σ(aᵢ - bᵢ)²)
```

- 范围：[0, +∞)，0 表示完全相同
- 同时考虑方向和大小
- 当向量已归一化时，L2 距离与余弦相似度等价

### 内积（Dot Product）

```
A·B = Σ(aᵢ × bᵢ)
```

- 如果向量已归一化，等价于余弦相似度
- 未归一化时同时反映相似度和向量大小（可用于推荐系统中"相似且重要"的场景）

## 近似最近邻搜索（ANN）

精确的 K 近邻搜索（KNN）需要计算查询向量与所有 N 个向量的距离，复杂度 O(Nd)。十亿级数据下不可行。

ANN 用少量精度损失换取数量级的速度提升。

### HNSW（Hierarchical Navigable Small World）

**原理：** 构建多层图，高层稀疏（长距离跳转），底层密集（精确搜索）。

```
Layer 2:  [A] ———————————— [G]        （少数节点，大步跳转）
Layer 1:  [A] —— [C] —— [E] —— [G]   （更多节点，中步搜索）
Layer 0:  [A]-[B]-[C]-[D]-[E]-[F]-[G] （所有节点，精确搜索）
```

**查询流程：**
```
1. 从最高层的入口节点开始
2. 在当前层贪心搜索最近邻
3. 到达局部最优后，下沉到下一层
4. 在底层找到最终 Top-K 结果
```

**优点：** 查询速度极快（O(log N)），召回率高（> 95%）
**缺点：** 内存开销大（需要存储图的边），构建时间长

### IVF（Inverted File Index）

**原理：** 先用 K-Means 将所有向量聚类到 C 个簇（Centroid），查询时只搜索最近的 nprobe 个簇。

```
构建：10 亿向量 → K-Means 聚类到 10,000 个簇
查询：找到最近的 10 个簇 → 只在这 10 个簇内搜索 → 候选范围缩小 1000 倍
```

**nprobe 的权衡：** nprobe 大 → 召回高但速度慢；nprobe 小 → 速度快但可能漏掉结果。

### Product Quantization（PQ）

**原理：** 将高维向量切分为 M 个子空间，每个子空间独立量化为一个码本 ID。

```
原始向量 (1536 维, 6KB):
[v1, v2, ..., v1536]

切分为 M=192 个子空间，每个 8 维:
[v1..v8] [v9..v16] ... [v1529..v1536]

每个子空间用 256 个聚类中心量化:
[code_1] [code_2] ... [code_192]

量化后: 192 bytes（压缩 32 倍）
```

**距离计算：** 用查找表（Look-Up Table），预计算查询向量与每个码本中心的距离，然后查表累加，极快。

### 组合方案

| 方案 | 内存 | 速度 | 召回率 | 适用场景 |
|---|---|---|---|---|
| Flat（暴力搜索） | 大 | 慢 | 100% | 小数据集基线 |
| HNSW | 大 | 极快 | > 95% | 中等规模，高性能要求 |
| IVF-PQ | 小 | 快 | 85-95% | 超大规模（十亿级） |
| IVF-HNSW-PQ | 中 | 极快 | > 90% | 大规模 + 高召回 |

## 向量搜索的工程挑战

**维度灾难：** 高维空间中所有点的距离趋于相等，ANN 索引效果下降。缓解：PCA 降维、Matryoshka Embedding。

**实时更新：** HNSW 支持增量插入，但删除和更新需要特殊处理（软删除 + 定期重建）。

**混合搜索：** 实际场景常需要"向量相似 + 条件过滤"（如"找相似的红色连衣裙，价格 < 200"）。需要索引支持元数据过滤。

**分布式：** 十亿级向量需要分片到多台机器，查询广播到所有分片，结果合并。

---

## 小结

> Embedding 和向量检索的核心是**距离度量选择和 ANN 索引算法**。面试高频题：余弦相似度 vs 欧氏距离的区别和适用场景、HNSW 的多层图搜索原理、IVF 的聚类加速原理、PQ 的压缩原理和距离计算方式、以及内存/速度/召回率的三角权衡。
