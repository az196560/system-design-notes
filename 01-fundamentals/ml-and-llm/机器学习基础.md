## 监督学习 vs 无监督学习 vs 强化学习

| 范式 | 数据 | 目标 | 典型算法 |
|---|---|---|---|
| 监督学习（Supervised） | 带标签（X, Y） | 学习 X→Y 的映射 | 线性回归、SVM、决策树、神经网络 |
| 无监督学习（Unsupervised） | 无标签（X） | 发现数据结构/分布 | K-Means、PCA、AutoEncoder |
| 强化学习（Reinforcement） | 环境反馈（奖励信号） | 最大化累计奖励 | Q-Learning、PPO、RLHF |
| 自监督学习（Self-Supervised） | 从数据自身构造标签 | 学习通用表示 | BERT（掩码预测）、GPT（下一词预测） |

## 损失函数（Loss Function）

### 回归任务

**MSE（均方误差）：** L = (1/n) Σ(yᵢ - ŷᵢ)²
- 对大误差敏感（平方惩罚），容易被异常值影响

**MAE（平均绝对误差）：** L = (1/n) Σ|yᵢ - ŷᵢ|
- 对异常值更鲁棒，但在零点不可导

**Huber Loss：** 小误差用 MSE，大误差用 MAE。兼具两者优点。

### 分类任务

**交叉熵（Cross-Entropy）：** L = -Σ yᵢ log(ŷᵢ)
- 二分类：L = -[y log(ŷ) + (1-y) log(1-ŷ)]
- 多分类：配合 Softmax 使用
- 为什么不用 MSE 做分类？因为 Sigmoid + MSE 的梯度在预测错误时很小（梯度消失），而交叉熵在预测错误时梯度大，收敛快。

**Focal Loss：** 在交叉熵基础上加权，降低易分样本的权重，聚焦难分样本。用于类别极度不平衡场景（如目标检测）。

## 优化器（Optimizer）

### SGD（随机梯度下降）

θ = θ - lr × ∇L

- 每次用一个 mini-batch 估计梯度
- 问题：学习率固定，所有参数用相同学习率；可能在鞍点附近震荡

### SGD + Momentum

v = β × v + ∇L
θ = θ - lr × v

- 引入动量（Momentum），累积历史梯度方向，加速收敛、减少震荡
- β 通常取 0.9

### Adam（Adaptive Moment Estimation）

结合 Momentum + RMSProp：
- 一阶矩（梯度均值 m）：跟踪梯度方向
- 二阶矩（梯度方差 v）：自适应调整每个参数的学习率
- 稀疏梯度的参数用更大学习率，频繁更新的参数用更小学习率

**AdamW：** Adam + Weight Decay。标准 Adam 的 L2 正则化与自适应学习率耦合，AdamW 解耦了 Weight Decay，训练效果更好。**大模型训练的标配优化器。**

| 优化器 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| SGD + Momentum | 泛化好，最终收敛效果常优于 Adam | 需要精细调学习率 | CV 任务、精调 |
| Adam/AdamW | 收敛快，自适应学习率 | 可能泛化不如 SGD | NLP、大模型预训练 |

## 学习率调度（Learning Rate Schedule）

- **Warmup：** 训练初期从很小的学习率线性增加到目标值，避免初始阶段梯度不稳定
- **Cosine Decay：** 学习率按余弦曲线逐渐衰减到接近 0
- **Warmup + Cosine Decay：** 大模型训练标配组合

---

## 正则化（Regularization）

防止过拟合的技术：

**L1 正则化：** 损失函数加 λΣ|θᵢ|。倾向于产生稀疏权重（部分权重为 0），起到特征选择作用。

**L2 正则化（Weight Decay）：** 损失函数加 λΣθᵢ²。倾向于让权重都较小但非零，防止某个权重过大。

**Dropout：** 训练时随机丢弃一部分神经元（如 p=0.1），迫使网络不依赖任何单一特征。推理时关闭 Dropout（或等价缩放）。

**Early Stopping：** 监控验证集 Loss，当验证集 Loss 不再下降时停止训练。

**Data Augmentation：** 通过数据变换（翻转、裁剪、噪声）增加训练数据多样性。

## 过拟合 vs 欠拟合

| 状态 | 表现 | 原因 | 解决方案 |
|---|---|---|---|
| 过拟合（Overfitting） | 训练集 Loss 低，验证集 Loss 高 | 模型太复杂 / 数据太少 | 正则化、Dropout、增加数据、减小模型 |
| 欠拟合（Underfitting） | 训练集和验证集 Loss 都高 | 模型太简单 / 训练不够 | 增大模型、延长训练、减少正则化 |

**偏差-方差权衡（Bias-Variance Tradeoff）：**
- 高偏差 = 欠拟合（模型太简单，抓不住规律）
- 高方差 = 过拟合（模型太复杂，记住了噪声）

## 评估指标

### 分类

| 指标 | 公式 | 含义 |
|---|---|---|
| Accuracy | (TP+TN) / 总数 | 整体正确率（类别不平衡时不可靠） |
| Precision | TP / (TP+FP) | 预测为正的里面有多少真正 |
| Recall | TP / (TP+FN) | 真正为正的里面找回了多少 |
| F1 | 2 × P × R / (P + R) | Precision 和 Recall 的调和平均 |
| AUC-ROC | ROC 曲线下面积 | 模型区分正负样本的能力，不受阈值影响 |

**Precision vs Recall 的权衡：**
- 医疗诊断：Recall 更重要（不能漏诊）
- 垃圾邮件：Precision 更重要（不能误杀正常邮件）

### 回归

| 指标 | 含义 |
|---|---|
| MSE / RMSE | 平均平方误差（对大误差敏感） |
| MAE | 平均绝对误差（对异常值鲁棒） |
| R² | 模型解释了多少数据方差（1 最好，0 最差） |

---

## 小结

> ML 八股的核心是**损失函数、优化器、正则化、评估指标**四大模块。面试时常考：交叉熵为什么比 MSE 适合分类、Adam vs SGD 的区别、L1 vs L2 正则化的效果差异、过拟合的诊断和解决方案、Precision vs Recall 的场景选择。
